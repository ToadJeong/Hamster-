# 푸시 알림 시나리오

원칙 5에 따라, 등록된 시나리오만 발송한다.
시나리오 추가는 PR로 합의한다.

## 등록된 시나리오

### S1. 가이드에 새 댓글
- **트리거**: `comments` INSERT 시
- **대상**: 가이드 작성자 (회원에 한함, 익명 작성자는 알림 대상 아님)
- **빈도**: 가이드당 하루 최대 3건으로 제한
- **본문**: `"{commenter_name}님이 내 가이드에 댓글을 남겼어요"`
- **딥링크**: `/guides/{id}`
- **구현 위치**: Supabase Trigger → Edge Function → FCM/APNs

### S2. 운영자 공지
- **트리거**: 관리자가 `/admin`에서 “공지 푸시 발송” 버튼 사용 시
- **대상**: 알림 동의한 모든 사용자
- **빈도**: 월 최대 1회 (정책)
- **본문**: 관리자가 입력 (최대 80자)
- **딥링크**: `/` 또는 `/guides/{id}`
- **구현 위치**: `/admin/announce` (v2)

### S3. 좋아요 50개 도달
- **트리거**: `likes` INSERT 후 count == 50일 때 1회
- **대상**: 작성자 (회원에 한함)
- **본문**: `"내 가이드가 사랑받고 있어요 🎉 좋아요 50개 돌파!"`
- **딥링크**: `/guides/{id}`

## 금지

- 마케팅 알림 (할인, 이벤트 광고)
- “이 콘텐츠 어때요?” 류의 추천 알림
- 같은 사용자에게 1시간 내 2건 이상의 같은 종류 알림

## 토글

사용자 설정에서 시나리오별로 ON/OFF 가능해야 한다 (v2 예정).
- 댓글 알림: 기본 ON
- 좋아요 도달 알림: 기본 ON
- 운영자 공지: 기본 ON (오프 가능)
